# Настройка автоматического запуска в Планировщике задач Windows 11

Инструкция по настройке регламентного фонового запуска скрипта через встроенный Планировщик задач (Task Scheduler).

## Предварительные требования
Перед настройкой убедитесь, что:
1. Создана рабочая директория приложения (например, `C:\TestUserEnv`).
2. Внутри рабочей директории развернуто виртуальное окружение и установлен wheel приложения (`C:\TestUserEnv\.venv`).
3. В корне рабочей директории создан и заполнен файл `.env`.

---

## Пошаговая настройка Планировщика

### Шаг 1. Открытие диспетчера
1. Нажмите клавиши `Win + R` на клавиатуре.
2. Введите `taskschd.msc` и нажмите `Enter`. Откроется окно **«Планировщик задач»**.

### Шаг 2. Создание базовой задачи
1. В правой панели **«Действия» (Actions)** нажмите **«Создать простую задачу...» (Create Basic Task...)**.
2. В открывшемся мастере укажите параметры:
   * **Имя (Name):** `7rights_api_integration` (или любое понятное имя).
   * **Описание (Description):** Автоматическая обработка почты и создание RFQ через API 7rights.
3. Нажмите **«Далее» (Next)**.

### Шаг 3. Настройка триггера (Расписание)
1. На этапе **«Триггер» (Trigger)** выберите частоту запуска (например, **«Ежедневно» / Daily**). Нажмите **«Далее»**.
2. Установите время начала работы скрипта и интервал повторения (1 день). Нажмите **«Далее»**.

### Шаг 4. Настройка действия
1. На этапе **«Действие» (Action)** выберите **«Запуск программы» (Start a program)**. Нажмите **«Далее»**.
2. Внимательно заполните поля формы запуска:
   * **Программа или сценарий (Program/script):** Укажите прямой абсолютный путь к исполняемому файлу вашей точки входа внутри виртуального окружения `.venv`, например.
     ```text
     C:\TestUserEnv\.venv\Scripts\run-api-integration.exe
     ```
   * **Аргументы (Add arguments):** Оставьте поле *пустым*.
   * **Рабочая папка (Start in):** Укажите абсолютный путь к папке, где лежит ваш `.env` файл. Это критически важно для работы Pydantic и сохранения логов/загрузок.
     ```text
     C:\TestUserEnv
     ```
3. Нажмите **«Далее»**, а затем **«Готово» (Finish)**.

---

## Важные финальные донастройки (Скрытый запуск и частота)

После создания задача появится в общем списке по центру экрана. Нам нужно настроить её на регулярный запуск в течение дня и скрыть появление черных окон консоли.

1. Найдите задачу `7rights_api_integration` в списке, нажмите по ней правой кнопкой мыши и выберите **«Свойства» (Properties)**.
2. На вкладке **«Общие» (General)**:
   * В блоке *«Параметры безопасности»* выберите пункт **«Выполнять вне зависимости от регистрации пользователя» (Run whether user is logged on or not)**. 
   * *Зачем:* Это скроет появление раздражающих черных окон терминала на экране пользователя каждые N минут. Скрипт будет незаметно работать в фоновом режиме.
3. На вкладке **«Триггеры» (Triggers)**:
   * Выделите ваш триггер и нажмите **«Изменить...» (Edit...)**.
   * Поставьте галочку напротив пункта **«Повторять задачу каждые:» (Repeat task every:)** и выберите или впишите вручную нужный интервал (например, `15 минут` или `1 час`).
   * В поле **«в течение:» (for a duration of)** выберите **«Бесконечно» (Indefinitely)**.
   * Нажмите **«ОК»**.
4. Нажмите **«ОК»** в главном окне свойств. Система Windows попросит вас ввести пароль от вашей учетной записи Windows для подтверждения фонового запуска.

---

## Как проверить работу?

1. В списке задач нажмите правой кнопкой мыши на `7rights_api_integration` и выберите **«Запустить» (Run)**.
2. Перейдите в папку `C:\TestUserEnv`.
3. Убедитесь, что в папке:
   * Появился обновляемый файл `pipeline.log`.
   * Создалась или обновилась папка `downloads/`.
   * В логах отсутствуют ошибки (строки со статусом `[ERROR]` или `Traceback`).
